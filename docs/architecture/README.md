```
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   Node 1      │    │   Node 2      │    │   Node 3      │
│ ┌───────────┐ │    │ ┌───────────┐ │    │ ┌───────────┐ │
│ │WebSocket  │ │    │ │WebSocket  │ │    │ │WebSocket  │ │
│ │  Manager  │ │◄──►│ │  Manager  │ │◄──►│ │  Manager  │ │
│ └─────┬─────┘ │    │ └─────┬─────┘ │    │ └─────┬─────┘ │
│       │       │    │       │       │    │       │       │
│ ┌─────▼─────┐ │    │ ┌─────▼─────┐ │    │ ┌─────▼─────┐ │
│ │Redis Pub/ │ │◄──►│ │Redis Pub/ │ │◄──►│ │Redis Pub/ │ │
│ │   Sub     │ │    │ │   Sub     │ │    │ │   Sub     │ │

        └─────────────────────┼─────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │     Redis       │
                    │   (Pub/Sub)     │
                    └─────────────────┘
```

### 核心组件

#### 1. Redis Pub/Sub 模块

**频道命名规范**:
```
{prefix}:room:{room_id}     # 房间广播频道
{prefix}:config:sync        # 配置同步频道
{prefix}:stream:audit       # 审计日志流
```

**消息流向**:
```
用户发送消息 → Node 1 WebSocket
                    │
        ┌───────────┼───────────┐
        │           │           │
        ▼           ▼           ▼
   本地广播    Redis 发布    数据库写入
   (Node 1)   (capella:room:id)
                    │
                    ▼
              Redis Pub/Sub
                    │
        ┌───────────┴───────────┐
        │                       │
        ▼                       ▼
   Node 2 订阅              Node 3 订阅
        │                       │
        ▼                       ▼
   本地广播                 本地广播
```

#### 2. 节点标识

每个节点启动时生成唯一的 `node_id`，用于：
- 避免消息循环广播
- 追踪消息来源
- 统计各节点负载

```rust
pub struct RoomBroadcastMessage {
    pub node_id: String,        // 发送节点标识
    pub room_id: Uuid,          // 房间 ID
    pub message: String,        // 消息内容
    pub exclude_user: Option<Uuid>,
    pub timestamp: DateTime<Utc>,
}
```

### 部署模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| 单节点 | 开发测试、小规模部署 | 简单、低延迟、无需 Redis |
| 多节点 | 生产环境、高可用 | 支持水平扩展、需要 Redis |

### 环境变量配置

```bash
# 启用 Redis 分布式支持
REDIS_ENABLED=true
REDIS_URL=redis://127.0.0.1:6379
REDIS_POOL_SIZE=10
REDIS_CHANNEL_PREFIX=capella
```

## 数据库优化 (阶段 8.6)

### 问题背景

**问题 1: 数据库写入压力**
- 审计日志高并发写入直接打向 PostgreSQL
- 万级 QPS 下批量写入可能阻塞主业务

**问题 2: 配置热更新同步**
- 配置变更仅单节点生效
- 多节点部署时配置不一致

### 解决方案

#### 1. Redis Stream 异步写入

```
┌─────────────┐     ┌─────────────┐     ┌─────────────────┐
│  业务逻辑    │────►│ Redis Stream │────►│  Consumer Group │
│  (Audit Log)│     │(capella:   │     │  (多节点负载均衡) │
└─────────────┘     │ stream:audit)│     └────────┬────────┘
                    └─────────────┘              │
                                                  ▼
                                         ┌─────────────────┐
                                         │  批量写入        │
                                         │ PostgreSQL      │
                                         └─────────────────┘
```

**流程说明**:
1. 业务逻辑将审计日志写入 Redis Stream
2. Consumer Group 从 Stream 消费消息
3. 批量写入 PostgreSQL（可配置批量大小）
4. ACK 确认后消息从 Pending List 移除

**降级机制**:
- Redis 不可用时，直接写入本地 Buffer 并定时刷盘
- Consumer 故障时，Pending List 重试机制
- 死信队列（DLQ）处理无法消费的消息

#### 2. 配置热更新同步

```
Node 1 修改配置
      │
      ▼
┌─────────────┐
│  ConfigManager│
│  (写入 DB)   │
└──────┬──────┘
       │
       ▼
┌─────────────┐     Redis Pub/Sub      ┌─────────────┐
│ Redis 发布   │ ─────────────────────► │  Node 2 订阅  │
│(config:sync)│                        │  本地重载     │
└─────────────┘                        └─────────────┘
                                       ┌─────────────┐
                                       │  Node 3 订阅  │
                                       │  本地重载     │
                                       └─────────────┘
```

**同步延迟**: < 100ms（内网环境）

### 配置项

```bash
# Redis Stream 配置
REDIS_STREAM_MAX_LEN=100000           # Stream 最大长度
REDIS_CONSUMER_BATCH_SIZE=100         # 批量消费大小
REDIS_CONSUMER_POLL_INTERVAL_MS=1000  # 消费间隔（毫秒）
REDIS_CONFIG_SYNC_ENABLED=true        # 启用配置同步
```

## 安全加固 (阶段 8.7)

### IP 黑白名单系统

#### 架构设计

```
客户端请求
     │
     ▼
┌─────────────────┐
│   负载均衡器     │
│ (X-Forwarded-For)│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  IP 安全检查     │
│  - 白名单模式检查 │
│  - 黑名单匹配    │
│  - CIDR 范围匹配 │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
  允许       拒绝
    │         │
    ▼         ▼
  正常处理   记录审计日志
            返回 403
```

#### 内存缓存机制

```
┌─────────────────────────────────────────┐
│           IP 安全检查流程                │
├─────────────────────────────────────────┤
│                                         │
│  请求到达 ──► 内存缓存检查 ──► 命中？    │
│                  │              │       │
│                  │ 否           │ 是    │
│                  ▼              ▼       │
│            数据库查询 ◄────── 返回结果   │
│                  │                      │
│                  ▼                      │
│            更新内存缓存                  │
│                  │                      │
│                  ▼                      │
│            返回检查结果                  │
│                                         │
└─────────────────────────────────────────┘

缓存刷新周期: 60 秒
```

#### CIDR 支持

支持单 IP 和 CIDR 范围：

```
单 IP:     192.168.1.100
CIDR:      192.168.1.0/24
IPv6:      2001:db8::/32
```

使用 `ipnet` crate 进行准确的 CIDR 验证和匹配。

#### 工作模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 黑名单模式 | 阻止特定 IP，其他允许 | 常规防护 |
| 白名单模式 | 仅允许特定 IP，其他阻止 | 高安全环境 |

#### WebSocket 层防护

```rust
// WebSocket 连接建立时进行 IP 检查
async fn ws_handler(
    ws: WebSocketUpgrade,
    ConnectInfo(addr): ConnectInfo<SocketAddr>,
    headers: HeaderMap,
    State(state): State<AppState>,
) -> Result<Response, AppError> {
    // 获取真实 IP（考虑代理头）
    let ip = extract_real_ip(&headers, addr);
    
    // IP 安全检查
    if let Some(ip_security) = &state.ip_security {
        if !ip_security.check_ip(&ip).await? {
            // 记录审计日志
            audit_service.log_ip_blocked(&ip).await?;
            return Err(AppError::Forbidden("IP blocked".into()));
        }
    }
    
    // 继续 WebSocket 升级...
}
```

### 审计追踪

所有安全事件都记录到审计系统：

| 事件类型 | 说明 |
|----------|------|
| `IpBlocked` | IP 被阻止 |
| `IpAllowed` | IP 被允许 |
| `IpWhitelistModeEnabled` | 启用白名单模式 |
| `SecurityCheckFailed` | 安全检查失败 |
